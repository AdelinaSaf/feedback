# Contact Form API с AI-анализом

Бэкенд-сервис для обработки обращений с лендинга. Реализует полный цикл: прием данных, валидацию, AI-анализ тональности, сохранение в БД и асинхронную отправку email-уведомлений.

- **Swagger документация:** https://feedback-production-b7f2.up.railway.app/docs
- **Репозиторий:** https://github.com/AdelinaSaf/feedback

---

## 1. Как запустить проект локально

**Требования:** Python 3.12+, PostgreSQL (или SQLite).

1. Клонируйте репозиторий: `git clone https://github.com/AdelinaSaf/feedback.git && cd feedback`
2. Создайте и активируйте виртуальное окружение: `python3 -m venv venv` и `source venv/bin/activate` (или `venv\Scripts\activate` на Windows)
3. Установите зависимости: `pip install -r requirements.txt`
4. Создайте файл `.env` на основе `.env.example` и заполните его своими данными (ключи API, настройки SMTP, DATABASE_URL).
5. Запустите сервер: `uvicorn app.main:app --reload`
6. Откройте http://127.0.0.1:8000/docs для тестирования API.

## 2. Стек технологий

- **Backend:** Python 3.12, FastAPI, Uvicorn
- **База данных:** PostgreSQL, SQLAlchemy (async), asyncpg
- **Валидация:** Pydantic V2
- **AI:** Groq API (модель llama-3.3-70b-versatile) через библиотеку openai
- **Инфраструктура:** SlowAPI (rate limiting), модуль logging, асинхронная отправка почты
- **Деплой:** Railway

---

## 3. Архитектура

Проект построен по принципам **слоистой архитектуры**, адаптированной под FastAPI:

- **routers/** (Controllers): Принимают HTTP-запросы, валидируют данные через Pydantic, возвращают ответы. Не содержат бизнес-логики.
- **services/**: Содержат бизнес-логику. `contact_service` оркестрирует процесс: вызывает AI, сохраняет данные и ставит задачу на отправку письма.
- **repositories/**: Отвечают исключительно за CRUD-операции с БД. Это позволяет легко заменить БД без изменения бизнес-логики.
- **schemas/**: Pydantic-модели для DTO.
- **core/**: Конфигурация (pydantic-settings), логирование, глобальные обработчики ошибок и инициализация БД.

*Почему FastAPI, а не Django?* Для этой задачи требовался легковесный, нативно асинхронный фреймворк с автогенерацией OpenAPI и встроенной валидацией. Django стал бы излишним усложнением (overengineering) для одного основного эндпоинта.

## 4. Реализация API

- **POST /api/contact**: Основной эндпоинт. Принимает name, phone, email, comment.
  - *Валидация:* Строгая проверка типов и форматов (regex для телефона, EmailStr). При ошибке возвращается 422 Unprocessable Entity с описанием на русском языке.
  - *Обработка ошибок:* Глобальные exception handlers перехватывают неожиданные ошибки и возвращают 500 Internal Server Error без утечки деталей.
- **GET /api/health**: Проверка статуса приложения и подключения к БД.
- **GET /api/metrics**: Возвращает общее количество обработанных обращений из БД.

---

## 5. AI-интеграция

- **Инструмент:** Groq API (Llama 3). Выбран из-за высокой скорости и совместимости с SDK OpenAI.
- **Задача:** Анализ тональности комментария (возвращает: "позитивный", "негативный", "нейтральный").
- **Graceful Fallback:** Вызов AI обернут в try/except с таймаутом 5 секунд. Если API недоступен, сервис не падает. Он логирует ошибку, присваивает тональности значение "неизвестный" и успешно завершает запрос, гарантируя доступность формы в любых условиях.

## 7. Хранение данных и инфраструктура

- **База данных:** PostgreSQL с асинхронным драйвером asyncpg.
- **Логи:** Все запросы и ошибки записываются в файл app.log с ротацией.
- **Rate Limiting:** Реализован через slowapi (5 запросов в минуту на IP). Данные хранятся в памяти.
- **Email:** Отправка через BackgroundTasks, чтобы не блокировать ответ клиенту.
  *Важное примечание по инфраструктуре:* при деплое на бесплатные тарифы облачных провайдеров (Railway) исходящий SMTP-трафик на порты 587/465 иногда блокируется сетевыми политиками. Архитектура приложения предусматривает это: сбой отправки письма корректно перехватывается и логируется, но **не влияет на доступность API** (клиент гарантированно получает ответ 200 OK). Для production рекомендуется использовать HTTP-API сервисов вроде Resend или SendGrid.
